# Don't Repeat Yourself (DRY) Code Instructions

- Avoid duplicating code by abstracting common functionality into reusable functions, classes, or modules.
- When you find yourself copying and pasting code, consider whether it can be refactored to eliminate the duplication.
- Use inheritance, composition, or higher-order functions to share behavior between different parts of the codebase without copying code.
- When refactoring to eliminate duplication, ensure that the new abstraction is well-designed and does not introduce unnecessary complexity.
- All code should follow best practices and be as performant as possible, so when refactoring to eliminate duplication, also consider whether the new abstraction can improve code quality and performance.
